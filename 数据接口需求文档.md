# 天梯赛数据接口需求文档

> **需求方**：运营团队
> **用途**：用户运营数据看板自动更新
> **期望交付时间**：--

---

## 一、需求背景

运营团队需要实时监控天梯赛用户数据，用于：
- 用户分层运营（核心/活跃/潜力/流失风险用户）
- 活跃度分析
- 积分流转监控
- 球馆运营效果评估

目前每次需要手动导出Excel，效率低且数据滞后。希望通过API实现数据自动同步。

---

## 二、接口需求清单

### 接口1：用户天梯数据查询

**用途**：获取所有用户的天梯赛数据

**请求方式**：`GET /api/v1/ladder/users`

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页条数，默认100，最大500 |
| season_id | int | 否 | 赛季ID，不传则返回最新赛季 |
| rank_level | string | 否 | 段位筛选：king/diamond/platinum/gold/silver/bronze |
| province | string | 否 | 省份筛选 |

**返回字段**：

### 接口2：用户积分流水查询

**用途**：获取积分变动记录

**请求方式**：`GET /api/v1/ladder/points-records`

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页条数，默认100，最大500 |
| venue_id | int | 否 | 球馆ID筛选 |
| user_id | int | 否 | 用户ID筛选 |
| business_type | string | 否 | 业务类型：GAME_FINISH等 |
| start_date | string | 否 | 开始日期 YYYY-MM-DD |
| end_date | string | 否 | 结束日期 YYYY-MM-DD |

## 三、技术要求

### 1. 认证方式
- 建议使用 **API Key** 认证
- 请求头携带：`Authorization: Bearer {api_key}`

### 2. 限流策略
- 建议每分钟最多 60 次请求
- 看板数据每 5-10 分钟刷新一次即可

### 3. 数据权限
- 仅返回运营所需字段，不返回敏感信息
- 可按球馆维度授权数据访问

### 4. 响应格式
- 统一 JSON 格式
- 支持 CORS 跨域（看板为前端页面）

---

## 四、对接流程

1. **开发方提供**：
   - API接口地址
   - API Key
   - 接口文档（如有Swagger更佳）

2. **运营方配合**：
   - 提供测试反馈
   - 确认数据字段是否满足需求

3. **上线后**：
   - 看板自动调用接口更新数据
   - 无需人工导出
